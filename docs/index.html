hybrid CapabilityRouter

#The router

An AX-grounded spine holds a chain of pluggable connectors, ordered by (cost, −confidence). The cheapest connector that is confident and available wins. A connector never raises — on failure it returns a result and the router falls through to the next rung.

1
app_launch
NSWorkspace + URL scheme open. No TCC permission needed — the cheapest tier.
2
media_key
Auxiliary HID media keys (play / pause / mute / next). Reversible, no confirm.
3
menubar_ax
Drives the focused app's menu bar via Accessibility.
4
deixis_click
The point-and-say effector: AX press at the fused pointer — "play THIS", "move THAT there".
6
browser_tab
Browser tab control via warm osascript.
7
mcp
Any server in ~/.curby/mcp_servers.json becomes a router tool dynamically. Opt-in; zero servers configured by default.
8
intent (LLM)
LLM intent-parse seam.
9
agent_loop
Structured Claude tool-use loop — exposes every connector as a tool, composes multi-step plans, dispatches each call back through the router. Per-step confirm for irreversible actions.
10
agent_fallback
Unstructured last resort — shells to claude -p. Always available.
11
computer_use
Pixel-level vision fallback for AX-opaque UIs. Requires Screen Recording permission.

tiebreak: cheaper cost wins; equal cost → higher confidence wins. .name doubles as the mechanism tag in telemetry.

feedback layer

#JARVIS HUD

A single SessionPhase state machine (idle → listening → heard → understanding → planning → acting → done/error) drives every surface in unison — nothing updates independently.

VISUAL

  • Animated reticle ORB — audio-reactive ring during speech, thinking-spin while the router plans, lock-on brackets when the action is confirmed.
  • Live partial-transcript caption — streams the in-progress recognition text under the crosshair as you speak.
  • Ambient screen-edge glow — subtle colored pulse tied to the current phase; changes hue at each state transition.
  • Radial connector-chain diagnostic ring — shows which cost rungs were tried and which one won.
  • Command / undo history overlay — scrollable record of recent actions with undo handles.
  • Adaptive-ink legibility — HUD ink auto-adjusts contrast for readability over any background content.

CONSOLE + AUDIO

  • Frosted Console card — progressive disclosure: chosen mechanism → latency chip ("did it in N ms") → undo toast.
  • Latency chip — per-action wall-clock time shown inline in the console card after every execution.
  • Undo toast — one-tap undo surfaced immediately after any reversible action.
  • Sub-100 ms ack chime — on-device sound plays the moment the router starts; confirms the system heard you before the action fires.
  • Sentence-stream TTS — first sentence plays before the full agent response is ready; no long silence waiting for generation.

performance model

#Latency

Streaming STT partials feed a speculative rule-parse off the Speech queue so the router starts before the utterance ends. Short deterministic commands fire near-immediately via cheap rungs.

FAST PATH

  • Streaming STT partials kick off a speculative rule-parse before the phrase ends — the router has a candidate before you finish speaking.
  • Media, launch, and tab commands fire via rungs 1–6: no LLM in the loop, sub-second round-trip.
  • Sub-100 ms ack chime fires the moment the command is received, before execution completes.
  • Sentence-stream TTS plays the first sentence of any agent response immediately — no full-generation wait.

BARGE-IN & BUDGETS

  • Barge-in — a new utterance or an open-palm STOP gesture cancels any in-flight action immediately.
  • P95 SLO budgets — per-stage latency is tracked with rolling P95 targets; regressions surface in telemetry before they reach the user.
  • All timing is written to the per-utterance JSONL trace for offline analysis.

gesture event bus

#Iron-Man gestures

The hand-signal pointer stream feeds a gesture event bus with hysteresis and cooldown — so a brief hand tremor never fires an accidental command.

✊
pinch
Confirm the pending action — equivalent to clicking the overlay confirm button.
✋
open palm
STOP / barge-in — immediately cancels any in-flight action or agent step.
👋
swipe
Directional verb: left/right = prev/next; up/down = scroll. Direction is inferred from swipe vector.

hysteresis + cooldown: each gesture requires a sustained hold past a threshold and a quiet period before the next gesture can fire — accidental triggers are suppressed.

guarantees

#Safety & observability

SAFETY

  • Irreversible actions require an overlay confirm before executing.
  • Multi-step task engine — agent_loop presents each planned step with a per-step confirm gate; every executed step is logged to the undo ledger.
  • Undo ledger + session memory — SQLite-backed; the console card surfaces an undo toast after each action.
  • One-time execution-grant token vault — sensitive agent actions require a one-time grant and cannot be silently replayed.
  • Secure-Input detection blocks keystroke injection at password fields.
  • Every AX call is watchdog-wrapped — a timeout falls through, never hangs.
  • Connectors never raise; errors return in the result so routing degrades gracefully.

OBSERVABILITY

  • Per-utterance trace_id telemetry — each utterance gets a unique trace ID; all routing decisions are written to a JSONL log (OTel-GenAI-aligned).
  • Per-connector circuit breakers — a connector that fails repeatedly trips its breaker; the router skips it and degraded-mode routing continues with the remaining chain.
  • --check preflight — reports Microphone / Speech / Accessibility / Automation / Screen-Recording / agent / MCP status and opens the correct Settings pane for anything missing.
  • Each connector's .name is its mechanism tag in telemetry.
  • The dry-run audit JSON is an inspectable, diffable decision record.
  • The golden harness drives all 8 demos through the real CLI.
